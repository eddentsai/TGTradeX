# 資金費率空單策略（Funding Rate Short）

入口腳本：`main_funcing_watch.py`

每小時在資金費率結算前數秒，對費率最負的幣種做空，賺取費率結算收益。

---

## 策略邏輯

Binance 永續合約的資金費率每小時結算一次（每小時 00:00）。當費率為負值時，做空方可收取費用。本策略的核心流程：

1. **HH:59:00** — 掃描所有幣對的 Premium Index，篩選「即將結算且費率低於門檻」的幣種，取前 3 名
2. **HH:59:58** — 抓取 Top3 最新市價，記錄進場參考價
3. **HH:00:00** — 對 Top1 幣種送出 MARKET 空單，隨後掛止損/止盈

---

## 可調參數

| 參數 | 預設值 | 說明 |
|------|--------|------|
| `THRESHOLD_PCT` | `-0.5` | 資金費率篩選門檻（%），費率必須低於此值才入選 |
| `LEVERAGE` | `5` | 槓桿倍數 |
| `POSITION_RATIO` | `0.02` | 每次下單動用可用餘額的比例（2%） |
| `USE_TESTNET` | `False` | 是否使用 Binance 測試網 |

---

## 執行時序

```
HH:59:00  fetch_premium_index_all()
           get_top3_symbols_nearest_funding(threshold_pct)
           → 印出 Top3：symbol / fundingRatePct / nextFundingTime

HH:59:58  fetch_all_latest_prices_v2()
           → 記錄 top3_price_map[symbol] = price

HH:00:00  get_account() → 取 available_balance
           calc_order_qty() → 計算下單數量
           place_order(SELL MARKET) → 空單進場
           fetch_all_latest_prices_v2() → 觀測下單後價格漂移（不影響流程）
           place_sl_tp_orders() → 掛止損 / 止盈
           get_pending_positions() → 記錄倉位快照
```

---

## 倉位計算

### 下單數量

```
notional = available_balance × POSITION_RATIO × LEVERAGE
qty      = floor(notional / price_5958, qty_precision)
```

進場參考價固定使用 **59:58 的快照價**（方案 A），避免 00:00 送單時再打一次 REST 增加延遲。

### 止盈 / 止損

以 `price_5958` 為基準，`rate = abs(THRESHOLD_PCT) / 100`：

| 方向 | 計算式 | 邏輯 |
|------|--------|------|
| 止盈（TP） | `price_5958 × (1 − rate)` | 空單往下走，price 跌才獲利 |
| 止損（SL） | `price_5958 × (1 + rate / 2)` | 空單往上反彈半幅即止損 |

以 `THRESHOLD_PCT = -0.5%` 為例：

- 進場 `price_5958 = 100`
- TP = `100 × 0.995 = 99.50`（下跌 0.5%）
- SL = `100 × 1.0025 = 100.25`（上漲 0.25%）

## 下單流程細節

### 進場單

```python
place_order({
    "symbol": symbol,
    "side": "SELL",
    "orderType": "MARKET",
    "qty": str(qty),
    "tradeSide": "OPEN",
})
```

### 風控單

```python
place_sl_tp_orders(
    symbol=symbol,
    side="SELL",
    qty=str(qty),
    sl_price=sl_price,
    tp_price=tp_price,
)
```

### 延遲觀測

進場單 ACK 後記錄 `ack_delay_ms`，並額外打一次價格 API 計算 `drift_bps`（基點漂移），僅用於觀測，不影響交易。

---

## Top3 篩選條件

由 `BinanceExchange.get_top3_symbols_nearest_funding()` 執行：

- 資金費率 < `THRESHOLD_PCT`
- 下次結算時間屬於「最近一批」（即當前整點）
- 依費率由小到大排序，取前 3 名

若無符合條件的幣種，當輪跳過，不進場。

---

## Skip 條件

| 時間點 | Skip 原因 |
|--------|-----------|
| HH:59:00 | `top3` 為空（無符合費率門檻的幣種） |
| HH:59:58 | `fetch_all_latest_prices_v2()` 失敗 |
| HH:00:00 | Top1 無對應價格、`qty <= 0`、下單例外 |

## 交易後分析

每次成功下單後，從 log 找出以下三個欄位：

```
[WS_ORDER_FILL] ... | symbol=HIVEUSDT | status=FILLED | avgPrice=0.08662 | fillTime=2026-05-05 20:00:00.045000 +0800
[WS_RATE_CHECK] ... | rate_now=-1.1266% | ...
```

| 欄位 | 從哪裡拿 | 範例 |
|------|----------|------|
| `symbol` | `WS_ORDER_FILL` | `HIVEUSDT` |
| `avgPrice` | `WS_ORDER_FILL` | `0.08662` |
| `rate_now` | `WS_RATE_CHECK` | `-1.1266` |
| 結算時間 | `fillTime` 去掉秒以下，整點為準 | `2026-05-05T20:00:00+08:00` |

### 執行分析腳本

```bash
python analyze_funding_klines.py HIVEUSDT 2026-05-05T20:00:00+08:00 \
    --entry 0.08662 \
    --rate -1.1266
```

選填參數：

| 參數 | 預設值 | 說明 |
|------|--------|------|
| `--minutes` | `5` | 觀察幾分鐘 |
| `--factor` | `0.8` | TP_RATE_FACTOR |
| `--sl-pct` | `0.3` | SL 幅度（%） |
