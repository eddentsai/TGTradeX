# bn（主流幣，Binance，4h）

監控 BTC、ETH、SOL、BNB 四個主流幣
- 根據市場狀態自動切換策略：
- 上升/下降趨勢 → Fibonacci（趨勢回調/反彈位入場，雙向）
- 震蕩市場 → VWAP+POC（偏離均值做均值回歸，雙向）

# bu（山寨幣，Bitunix，15m）

- 自動掃描山寨幣，從 700+ 個合約中選出成交量前幾名
- 三個策略同時評估，至少 2/3 確認才開倉（Ensemble 投票）：
  - Fibonacci：找波段高低點，在回調/反彈的 fib 位入場
  - VWAP+POC：超買做空、超賣做多
  - Dip/Volume：急跌爆量做多（撿反彈）、急漲爆量做空（抓頂）
- 急漲 >40% 的幣會放行進入策略評估（可觸發做空），急跌 >40% 排除並冷卻 24h

# oi_ls_ratio（OI + 多空比，Bitunix，15m）

利用 Binance 公開 API 抓取 OI 與多空比，偵測軋空/軋多訊號。

## 進場條件（需全部滿足）

### 做多（多頭軋空）
- OI 連續上升 ≥ 1.5%（近 5 期）
- 多空比連續下降 ≥ 3.0%（空方加倉但 OI 升 → 空方將被軋）
- OI 與多空比各自單調性 ≥ 60%（至少 60% 的期間方向一致）
- RSI < 65（避免超買追多）
- 24h 漲幅 < +15%（防止追高，ENJUSDT 教訓）

### 做空（空頭軋多）
- OI 連續上升 ≥ 1.5%
- 多空比連續上升 ≥ 3.0%（多方加倉但 OI 升 → 多方將被軋）
- OI 與多空比各自單調性 ≥ 60%
- RSI > 35（避免超賣追空）
- 24h 跌幅 < -15%（防止空頭擁擠）

## 風控參數
- 止損 SL：信號價 ×（1 ± sl_pct）
- 止盈 TP：信號價 ×（1 ± tp_pct）
- 移動止損：從峰值回落 trail_pct 觸發
- SL 冷卻：止損後同幣種 2 小時不再進場（防 MYXUSDT 式反覆開關）
