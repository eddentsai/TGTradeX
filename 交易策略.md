# bn（Binance，4h） + bu（Bitunix，1h）— 共用 OI 背離做多策略

兩個服務現在都跑 `run_oi_long.py`，差別只在交易所與 K 線週期。

## 信號識別流程

1. **波動性掃描**：從所有合約中，按 24h 高低價振幅排序，取前 30 名
2. **OI 背離篩選（嚴格版）**：
   - OI 48h 增加 > +20%（資金悄悄佈局）
   - 同期價格變化 < ±3%（市場還沒反應）
3. **多空比確認**：近 5 期多空比下降 > 3%（進場資金多數在做空 → 軋空佈局）
4. **技術確認**：收盤 > EMA20 且 RSI < 70

# 原 oi_ls_ratio 邏輯記錄（Bitunix，15m）

利用 Binance 公開 API 抓取 OI 與多空比，偵測軋空/軋多訊號。

## 進場條件（需全部滿足）

### 做多（多頭軋空）
- OI 連續上升 ≥ 1.5%（近 5 期）
- 多空比連續下降 ≥ 3.0%（空方加倉但 OI 升 → 空方將被軋）
- OI 與多空比各自單調性 ≥ 60%（至少 60% 的期間方向一致）
- RSI < 65（避免超買追多）
- 24h 漲幅 < +15%（防止追高，ENJUSDT 教訓）

### 做空（空頭軋多）
- OI 連續上升 ≥ 1.5%
- 多空比連續上升 ≥ 3.0%（多方加倉但 OI 升 → 多方將被軋）
- OI 與多空比各自單調性 ≥ 60%
- RSI > 35（避免超賣追空）
- 24h 跌幅 < -15%（防止空頭擁擠）

## 風控參數
- 止損 SL：信號價 ×（1 ± sl_pct）
- 止盈 TP：信號價 ×（1 ± tp_pct）
- 移動止損：從峰值回落 trail_pct 觸發
- SL 冷卻：止損後同幣種 2 小時不再進場（防 MYXUSDT 式反覆開關）
